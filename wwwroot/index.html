API Reference & Integration Guide

REST API Endpoints

Other programs can query and control the streamer service by sending standard HTTP requests to the following REST API endpoints:

GET /api/status

Retrieves the running status of the server and all active camera streams (including resolutions, codecs, current FPS, and state).

{
  "serverRunning": true,
  "mediamtxRunning": true,
  "mediamtxPort": 8554,
  "cameras": [
    {
      "ipAddress": "192.168.1.101",
      "streamName": "cam1",
      "status": "Streaming",
      "fps": 29.9,
      "width": 1920,
      "height": 1080,
      "codec": "H264",
      "encoder": "NVENC"
    }
  ]
}
GET /api/logs

Retrieves the 200 most recent lines of console logs.

POST /api/control/start

Starts the MediaMTX process and triggers Pylon grabbing for all configured cameras.

POST /api/control/stop

Stops all camera capture loops and terminates the MediaMTX process cleanly.

POST /api/control/restart-camera?ip={camera_ip}

Restarts only the specified camera's capture and streaming pipeline (fails with 404 if the IP is not in configuration).

POST /api/config

Saves and updates the system configuration (`config.json`). Requires the new config object in the request body.

Receiving Video Streams

1. Standard RTSP stream (VLC, OpenCV, Python)

Connect using standard TCP transport layer. The format is:

rtsp://<C#_Server_IP>:8554/<RTSP_Stream_Path>

Python OpenCV Sample:

import cv2
cap = cv2.VideoCapture("rtsp://192.168.1.50:8554/cam1")
while cap.isOpened():
    ret, frame = cap.read()
    if ret:
        cv2.imshow('Live Stream', frame)
    if cv2.waitKey(1) & 0xFF == ord('q'):
        break
cap.release()
cv2.destroyAllWindows()

2. WebRTC / WHEP stream (Ultra-low latency web player)

Retrieve live low-latency video via WHEP client-side negotiation:

http://<C#_Server_IP>:8889/<RTSP_Stream_Path>
